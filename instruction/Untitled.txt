## 🔍 **问题分析**

从截图可以看出移动端存在以下问题：

1. **卡片内边距过大** - 移动端应该更紧凑
2. **标题和价格换行处理** - 长标题可能挤压价格显示
3. **标签排列优化** - 多个标签在移动端的排列
4. **整体高度优化** - 移动端卡片高度可以更灵活

## 📱 **移动端优化方案**## 🎯 **主要改进点**

### 1. **移动端布局优化**
- 减小内边距：从 20px 减少到 16px/14px
- 允许标题换行：移动端显示完整标题
- 优化标签间距：减少到 4px/3px
- 灵活的卡片高度：350px → 320px

### 2. **响应式文字大小**
- 标题：1.1rem → 1rem → 0.95rem
- 价格：1.25rem → 1.125rem → 1.1rem  
- 标签：0.75rem → 0.7rem → 0.65rem

### 3. **图片区域优化**
- 桌面端：200px 高度
- 移动端：160px → 140px 高度
- 节省垂直空间

### 4. **容器边距调整**
- 移动端边距从 16px 减少到 12px
- 为卡片内容留出更多空间

## 📝 **实施步骤**

1. **更新 CSS 文件**：
   将上面的优化样式添加到 `frontend-spa/src/styles/tokens.css` 中，替换现有的卡片相关样式

2. **清理冲突样式**：
   确保移除任何可能冲突的旧样式

3. **测试验证**：
   - 在不同屏幕尺寸下测试
   - 确认单列布局占满宽度
   - 验证卡片内容显示完整

这样优化后，移动端的商品卡片将更加紧凑美观，充分利用屏幕空间，同时保持良好的可读性。
